# QRDS/QOS Data Acquisition / Depth Expansion Plan

Sprint 9N converts observed data-depth gaps from the research-only evidence stack into a concrete dataset expansion plan.

It is intentionally blocked from operational use. It defines only research-data requirements: symbol coverage, row-depth targets, path patterns, lineage checks, timestamp checks, duplicate checks, null-rate checks, and refresh policy.

Safety state:

- App mode: `INTERACTIVE_RESEARCH_ONLY`
- No API key required
- No account connection required
- No orders
- No real capital
- No executable signal
- No portfolio decision

Default target: 5,000 rows per symbol for the current research interval.
